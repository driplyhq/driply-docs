# Структура backend (Driply)

Feature-based архитектура: каждый домен — отдельный модуль.  
HTTP-слой тонкий, бизнес-логика в `modules/`, общие утилиты в `shared/`.

```text
driply-backend/
│
├── alembic/                    # Миграции БД
├── uploads/                    # Локальные файлы (аватары), не в git
├── docker-compose.yml
├── Dockerfile
├── requirements.txt
│
└── app/
    ├── main.py                 # Точка входа FastAPI, middleware, static
    ├── config.py               # Настройки из .env
    │
    ├── db/                     # Подключение к PostgreSQL
    │   ├── base.py             # Declarative Base
    │   ├── session.py          # Async engine / session
    │   └── deps.py             # Depends(get_db)
    │
    ├── api/                    # HTTP-слой (только роуты)
    │   └── v1/
    │       ├── router.py       # Сборка всех роутеров + /health
    │       └── endpoints/
    │           ├── auth/       # POST /auth/register, /login, /logout, GET /me
    │           └── users/      # Профиль, подписки, список пользователей, аватар
    │
    ├── modules/                # Фичи / домены
    │   ├── auth/
    │   │   ├── service.py      # register, login, get_current_user
    │   │   └── schemas.py      # UserRegister, UserLogin, RegisterResponse
    │   ├── users/
    │   │   ├── models.py       # SQLAlchemy: User
    │   │   ├── schemas.py      # Profile, UserListItem, UserUpdate*
    │   │   └── service.py      # Профиль, список, аватар, update
    │   └── follows/
    │       ├── models.py       # SQLAlchemy: Follow
    │       ├── schemas.py      # FollowListItem
    │       └── service.py      # follow, unfollow, followers/following
    │
    └── shared/                 # Общее для всех модулей
        ├── schemas/
        │   └── common.py       # UploadFileResponse и т.п.
        └── storage/
            ├── s3.py           # S3 + локальный fallback
            └── media.py        # Валидация изображений
```

## Правила

| Слой | Что кладём |
|------|------------|
| `api/endpoints/` | Только роуты: валидация входа, вызов service, response_model |
| `modules/*/service.py` | Бизнес-логика, работа с БД |
| `modules/*/models.py` | SQLAlchemy-модели домена |
| `modules/*/schemas.py` | Pydantic-схемы домена |
| `shared/` | Storage, общие схемы, утилиты без домена |

## Планируемые модули (MVP)

| Модуль | Статус | Описание |
|--------|--------|----------|
| `auth` | ✅ | Регистрация, вход, сессия |
| `users` | ✅ | Профиль, аватар, бренды |
| `follows` | ✅ | Подписки |
| `catalog` | 🔜 | Вещи, категории, стили |
| `outfits` | 🔜 | Аутфиты, лента |
| `favorites` | 🔜 | Избранное |
| `admin` | 🔜 | Админ-панель |

## Импорт моделей в Alembic

В `alembic/env.py` импортируй все модели, чтобы autogenerate видел таблицы:

```python
from app.modules.users.models import User
from app.modules.follows.models import Follow
```
