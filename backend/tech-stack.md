# Обоснование выбора технологического стека

Выбранный стек оптимален для данного проекта, так как сочетает скорость разработки, масштабируемость и поддержку AI-функций.

---

**FastAPI** обеспечивает быстрый и асинхронный API-first backend, подходящий для социальной и рекомендательной системы.  
**PostgreSQL** используется как основная база данных благодаря надёжности и поддержке сложных связей между сущностями (пользователи, аутфиты, лайки, подписки).  
**SQLAlchemy + Alembic** обеспечивают удобную работу с данными и управляемые миграции.  
**session-based** дают безопасную и гибкую систему авторизации.  
**Redis** используется для кеширования и ускорения feed-логики и API.  
**Celery** позволяет выносить тяжёлые задачи (AI, обработка изображений) в фоновые процессы.  
**Cloud storage (S3-совместимое)** обеспечивает хранение медиа-контента и масштабируемость.  
**PyTorch + Diffusers** используются для AI-генерации образов и рекомендаций.  
**OpenCV + Pillow** отвечают за обработку изображений.  
**Uvicorn** обеспечивает стабильный запуск FastAPI с высокой производительностью.

---

# MVP стек (первая версия продукта)

Для первой версии приложения используется упрощённый стек, ориентированный на быстрый запуск и проверку гипотезы.

## Основной MVP стек:

- FastAPI — основной backend API
- PostgreSQL — хранение всех данных
- SQLAlchemy — работа с базой данных
- Alembic — миграции базы данных
- session-based — авторизация пользователей
- Uvicorn — запуск сервера
- Cloud storage (S3 / R2) — хранение изображений

---

## Исключено из MVP (добавляется позже):

- Redis (кеширование и ускорение системы)
- Celery (фоновые задачи и AI-обработка)
- OAuth2 (вход через Google / Apple)
- PyTorch / Diffusers (AI-генерация)
- OpenCV / Pillow (расширенная обработка изображений)

---

## Итог MVP архитектуры:

```text
Frontend
   ↓
FastAPI (backend)
   ↓
PostgreSQL (данные)
   ↓
S3 / R2 (изображения)
```

---
