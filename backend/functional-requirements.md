 # Функциональные требования MVP (Backend)

# 1. Система пользователей

## 1.1 Регистрация и авторизация

Backend должен обеспечивать:

- регистрацию пользователя;
- вход в аккаунт;
- session-based аутентификацию;
- выход из аккаунта.

// Выбоор сделан в пользу session-based, из за простоты и бензопасности метода, JWT излишен для нашего проект

---

## 1.2 Профиль пользователя

Пользователь должен иметь возможность:

- изменять имя профиля;
- загружать аватар;
- просматривать свои публикации;
- просматривать избранное.

---

# 2. Каталог одежды

## 2.1 Работа с одеждой

Backend должен:

- хранить данные о вещах;
- поддерживать категории одежды;
- хранить изображения вещей;
- хранить стили.

### Данные вещи:

- id;
- название;
- категория;
- изображения;
- стиль;

---

## 2.2 Фильтрация

Система должна поддерживать:

- фильтрацию по категории;
- фильтрацию по стилю;
- поиск по названию.

---

# 3. Free Mode (конструктор образов)

## 3.1 Свайп-механика

Backend должен:

- фиксировать лайки вещей;
- фиксировать дизлайки вещей;
- получить следующую вещь;
- вернуться к предыдущей вещи;
- сохранять историю свайпов.

---

## 3.2 Создание аутфитов

Пользователь должен иметь возможность:

- создавать аутфиты;
- сохранять аутфиты;
- публиковать аутфиты;
- удалять аутфиты.

### Аутфит должен содержать:

- список вещей по определенным правилам с изображением вещи;
- описание;
- автора;
- дату сохранения;
- дату публикации.

---

## 3.3 Работа с изображениями

Backend должен:

- принимать изображения;
- хранить изображения.

---

# 4. Outfit Feed (лента аутфитов)

## 4.1 Лента аутфитов

Backend должен:

- отображать опубликованные аутфиты;
- сортировать аутфиты по дате;
- отображать популярные аутфиты.

---

## 4.2 Взаимодействие с аутфитами

Пользователь должен иметь возможность:

- лайкать аутфиты;
- сохранять аутфиты;
- просматривать профили авторов.

---

# 5. Социальные функции

## 5.1 Подписки

Backend должен поддерживать:

- подписку на пользователей;
- список подписчиков;
- список подписок.

---

# 6. Административная панель

Администратор должен иметь возможность:

## 6.1 Управление публикациями

- удалять публикации.

## 6.2 Управление пользователями

- просматривать список пользователей;
- искать пользователей;
- просматривать профили;
- блокировать и разблокировать аккаунты;
- удалять пользователей.

## 6.3 Управление одеждой

- загружать изображения одежды;
- указывать название вещи;
- назначать категорию;
- добавлять описание и теги;
- редактировать и удалять элементы одежды.

## 6.3 Управление стилями и категориями

- добавлять стиль;
- удалять стиль;
- добавлять категорию;
- удалять категорию.


---

# Нефункциональные требования MVP

# 1. Производительность

Система должна:

- обеспечивать стабильную работу при одновременной работе пользователей;
- обеспечивать быструю загрузку ленты;
- обеспечивать быстрый отклик API.

---

# 2. Безопасность

Система должна:

- использовать HTTPS;
- шифровать пароли;
- обеспечивать session-based-аутентификацию;
- защищать API.

---

# 3. Хранение данных

Система должна:

- поддерживать хранение изображений;
- поддерживать облачное хранилище медиа;
- обеспечивать быстрый доступ к данным.

---

# 4. API

Backend должен:

- предоставлять REST API;
- поддерживать документацию Swagger/OpenAPI.

---

# 5. Поддержка и расширяемость

Система должна:

- иметь модульную архитектуру;
- поддерживать дальнейшее расширение проекта.
