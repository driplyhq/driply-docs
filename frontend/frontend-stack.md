**Frontend стек для MVP**  
**1\. Framework**  
React.js \+ Next.js  
**Почему:**  
React — лучший выбор для:   
сложного интерактивного UI (конструктор аутфитов)   
переиспользуемых компонентов   
Next.js даёт:   
1)готовую структуру проекта   
2)Легкий routing   
**2\. Язык**  
TypeScript \- типизаия над javascript (интепретируемый и работает на v8 machine)   
**Почему**  
У нас сложные сущности:   
1)Outfit   
2)Clothing   
3)User   
4)Много логики (конструктор, состояния)   
TypeScript:  
1)уменьшает баги  

**3\. Работа с API**  
Axios \+ React Query (TanStack Query)  
**Почему:**  
Axios:  
1)удобный HTTP клиент   
2)interceptors (авторизация)   
React Query:  
1)кэширование данных   
2)автоматические обновления   
3)Легий loading / error состояния  

**4\. UI библиотека**  
Tailwind CSS \+ Headless UI   
**Почему:**  
Tailwind:  
1)быстрое прототипирование (MVP)   
2)контроль дизайна (важно для нашего продукта)   
3)лёгкий вес   
Headless UI:  
1)готовые интерактивные компоненты (modal, dropdown)   
2)без навязанного дизайна 

**5\. Анимации и жесты**  
Framer Motion  
**Почему:**  
1)плавные анимации (60 FPS)   
2)drag & swipe   
идеально для:   
1)Tinder-like свайпов   
2)анимаций лайков   
3)переходов 

**6\. State Management**  
Zustand  
**Почему:**  
легко управлять:   
1)состоянием конструктора   
2)пользователем   
3)UI состояниями 

**7\. Структура проекта**  
Feature-based   
**Почему:**  
проект \= не просто UI, а:   
1)конструктор   
2)соцсеть   
3)лента
